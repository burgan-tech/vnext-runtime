This documentation set explains **what the vNext Workflow Engine is**, **what business value it provides**, and **how it integrates** with enterprise systems—using business language and outcomes rather than developer-level API/reference detail.

- Business stakeholders evaluating workflow orchestration for their organization
- Product owners and solution architects shaping process automation initiatives
- Technical leadership aligning platform capabilities with operational needs

## How to use this documentation

- Start with the **Executive Summary** for a platform overview and value proposition
- Use **Platform Capabilities** and **Core Features** to map needs to capabilities
- Use **Integration Capabilities** and **Operational Features** to assess enterprise readiness
- Use **Use Cases** for business scenarios and adoption patterns
